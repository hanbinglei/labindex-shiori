# LabIndex Shiori — 研究资料横断检索系统

## 基本信息

| 字段 | 值 |
|------|-----|
| 项目名称 | LabIndex Shiori |
| 版本 | MVP 1.0 |
| 目标环境 | Windows（测试期）→ 共享常驻电脑 |
| 开发模式 | Vibe Coding（Hermes 执行 + Claude Opus 4.8 审查） |

## 目的

在松尾研究室 TeraStation NAS 上构建一个「像内置功能」的研究资料检索系统，让研究者能快速横断检索各类研究资料（论文、图纸、实验数据等），无需手动翻找文件夹。

## 核心问题

NAS 上积累了多年研究资料（PDF 论文、Word 报告、Excel 数据、CAD 图纸等），但缺乏统一的检索手段。研究者只能通过文件夹层级手动翻找，效率低、易遗漏。

## 产品边界

- ✅ 跨格式全文检索（PDF/docx/xlsx + 图纸元信息）
- ✅ 日语优先的多语言支持
- ✅ 增量扫描，只处理变更文件
- ✅ 人工纠错 overlay 机制（重扫不覆盖人工修正）
- ✅ 子主题自动归类 + 候选生成
- ✅ Timeline 年度脉络视图
- ✅ 关联视图（共同关键词）
- ✅ 命令行触发，供日后 Hermes 调度

## 明确非目标

- ❌ 不是全研究室部署（当前仅扫描 Han_Jingpeng 共享）
- ❌ 不修改 NAS 上的原始文件（只读挂载）
- ❌ 不调用任何外部 API / 云端服务（完全离线）
- ❌ 不支持多用户并发/权限系统（单人 MVP）
- ❌ 不是文件管理器/网盘
- ❌ 不处理音视频文件内部文本

## 技术栈

| 层 | 选型 | 理由 |
|----|------|------|
| 检索引擎 | Meilisearch | 本地离线、日语分词、轻量 |
| 后端/扫描 | Python 3.11+ | 生态丰富、跨平台 |
| 数据层 | SQLite | 单文件零运维、overlay 分离天然 |
| 前端 | Web UI（Flask/FastAPI + 原生 HTML/JS） | localhost 访问、零额外依赖 |
| 配置 | YAML | 人类可读、注释友好 |

## 架构原则

- **计算与存储分离**：文件留在 NAS，系统跑在一台电脑上
- **扫描范围配置化**：SCAN_ROOTS 改配置即可扩展
- **分类体系配置化**：大主题/子主题为可扩展数据
- **Overlay 优先**：人工修正存独立表，重扫不覆盖
- **多语言可维护**：文案全部走 i18n 文件，不写死代码里

## 预期产物

1. 增量扫描引擎
2. 多格式解析器（PDF/docx/xlsx + 图纸元信息）
3. Meilisearch 索引 + 日语分词检索
4. 子主题自动归类
5. Overlay 人工纠错界面
6. 多语言 Web UI（Timeline + 关联视图 + 检索筛选）
7. Hermes 调度用 CLI 入口
